# 🔮 易经寻物 Skill

## 概述

本 Skill 是易经寻物推演的**调度中心**，负责按需加载知识库并编排推演工作流。

核心目标：通过中国传统易经（梅花易数）原理，根据物品信息、丢失时间或随机数字，推演遗失物品的大致方位、所处环境及找回概率。

## 核心原则

### 先起卦、后分析、再建议

易经寻物采用"先起卦、后分析、再建议"的标准工作流：

1. **收集信息** - 了解丢失物品名称、材质、丢失时间、位置信息、随心数字
2. **起卦推演** - 根据时间或报数计算本卦、互卦、变卦
3. **体用分析** - 确定体卦（失主）和用卦（失物）
4. **方位环境推断** - 结合八卦方位、卦象特征、位置信息推断物品位置
5. **概率判断与建议** - 根据五行生克关系、位置环境分析判断寻回概率，给出寻找建议

## 三大核心工作流

### @工作流 1：信息收集与起卦

**依赖知识库**：`references/theory/MEIHUA_YISHU.md`

在推演之前，必须先收集以下信息。采用**分阶段收集**方式，每阶段提供选项供用户选择：

#### 阶段一：物品基本信息（必填）

1. **物品名称** - 丢失的具体物品（如：钥匙、钱包、手机）
2. **物品材质** - Agent 会根据物品名称自动推断材质，无需用户手动选择
   - 示例：用户说"丢了金戒指" → 自动推断材质为"金属"
   - 可选材质：金属/木质/植物/水/液体/火/电子产品/土/陶瓷/矿物/布料/纸张/不确定

#### 阶段二：时间与位置信息（必填）

3. **丢失时间** - 发现丢失的具体时间（格式：YYYY-MM-DD HH:mm）
   - 如果用户记不清丢失时间，默认使用当前系统时间起卦
   - 在结果中标注"起卦时间基于当前时刻"
4. **最后见到时间** - 最后一次见到物品的时间（格式：YYYY-MM-DD HH:mm）
5. **最后见到位置** - 最后一次见到物品的地点（如：家中玄关、公司工位）
6. **当前位置** - 用户当前所在位置（如：家中客厅、公司会议室）

#### 阶段三：位置环境详情（推荐填写）

7. **房间类型** - 最后见到物品的房间类型
   - 玄关/客厅/卧室/厨房/洗手间/书房/阳台/餐厅/储物间/走廊
8. **附近物品** - 物品附近的物件（如：鞋柜、钥匙挂钩、包包）
9. **家具** - 房间内的家具（如：鞋柜、换鞋凳、穿衣镜）
10. **近期活动** - 最后见到物品前后的活动（如：早上出门时拿了钥匙）
11. **已搜索区域** - 已经寻找过的地方（如：客厅沙发、卧室床头柜）
12. **大门朝向** - 你家大门大概朝哪个方向？（如：朝南、朝北、朝东等）
    - 说明：大门朝向会影响房间八卦映射，不知道也没关系

#### 阶段四：起卦信息（可选）

13. **随心数字** - 让用户深呼吸，随心报出 2-3 个数字（用于报数起卦）

#### 参考点选择

- **方位基准点**：可选择以"当前位置"或"最后见到位置"为基准进行方位推演
  - `current`：以当前所在位置为基准
  - `last_seen`：以最后见到物品的位置为基准（推荐）

#### 起卦方式选择

- **时间起卦**（默认）：使用丢失时间的年月日时计算上卦、下卦、动爻
- **报数起卦**（优先）：使用用户提供的数字计算上卦、下卦、动爻

> 详细起卦方法见 `@读取: references/theory/MEIHUA_YISHU.md`

### @工作流 2：体用分析与方位推断

**依赖知识库**：`references/theory/BAGUA_PROPERTIES.md`

根据起卦结果进行体用分析和方位推断：

1. **体用确定** - 根据动爻位置确定体卦（失主）和用卦（失物）
2. **方位推断** - 用卦所在八卦方位即为物品大致方向
3. **环境推断** - 结合互卦和变卦的卦象特征推断物品所处环境
4. **位置环境分析** - 结合用户提供的房间类型、附近物品、家具等信息，给出更精准的位置建议
5. **二进制爻象计算** - 使用位运算将六爻转换为二进制数值，快速定位卦象特征
   - 阳爻为 1，阴爻为 0，自下而上排列形成 6 位二进制数
   - 通过位运算快速计算互卦、变卦的卦象编号

> 详细八卦属性见 `@读取: references/theory/BAGUA_PROPERTIES.md`

### @工作流 3：五行生克与寻回建议

**依赖知识库**：`references/theory/WUXING_RELATIONS.md`

根据体卦和用卦的五行关系判断寻回概率：

1. **五行关系判断** - 分析体卦五行与用卦五行的生克关系
2. **概率评估** - 根据生克关系给出寻回概率（极高/较高/中等/较低/很低）
3. **位置环境评分** - 根据提供的位置信息完整度调整置信度
4. **动态加权融合** - 结合时间衰减、信息完整度、卦象稳定性动态调整权重
   - 时间衰减：丢失时间越长，寻回概率权重越低
   - 信息完整度：提供的信息越完整，置信度越高
5. **寻找建议** - 结合方位、环境、概率、位置信息给出具体寻找建议

> 详细五行生克关系见 `@读取: references/theory/WUXING_RELATIONS.md`

## 使用方法

### 完整推演流程

用户需要提供以下信息：

```
请帮我寻找丢失的物品：

【物品信息】
- 物品名称：家门钥匙
- 物品材质：金属（自动推断）
- 物品描述：一串有三把钥匙的钥匙串，有一个红色钥匙扣
- 重要程度：高

【时间与位置】
- 丢失时间：2026-05-14 14:30（记不清时使用当前时间）
- 最后见到时间：2026-05-14 08:00
- 最后见到位置：家中玄关
- 当前位置：家中客厅
- 方位基准点：最后见到位置

【环境详情】
- 房间类型：玄关
- 附近物品：鞋柜、钥匙挂钩、包包
- 家具：鞋柜、换鞋凳、穿衣镜
- 近期活动：早上出门时拿了钥匙
- 已搜索区域：客厅沙发、卧室床头柜
- 大门朝向：朝南

【起卦信息】
- 随心数字：7, 2
```

### 快速推演模式

如果用户只想快速知道方位：

```
快速寻物：
物品：钱包
材质：布料（自动推断）
丢失时间：2026-05-14 20:00（记不清时使用当前时间）
最后见到位置：公司工位
```

### 信息不足处理流程

如果用户只提供部分信息：

**第一步：先进行基础推演**
- 使用已有信息进行起卦和推演

**第二步：分阶段询问缺失信息**

- **第一阶段**："请告诉我丢失的是什么物品？"
- **第二阶段**："大概什么时候发现丢失的？最后在哪里见到它的？"
- **第三阶段**："能描述一下那个房间吗？（玄关/客厅/卧室等）附近有什么物品？你家大门大概朝哪个方向？"
- **第四阶段**："已经找过哪些地方了？"
- **第五阶段**（可选）："请随心报出两个数字，助我起卦"

**第三步：用户愿意补充**
- 根据补充信息重新推演
- 输出完整推演结果

**第四步：用户不想补充**
- 基于已有信息进行推演
- 在结果中标注"部分信息基于推测"
- 仍提供寻找建议，但建议补充信息以获得更精准的结果

## 风险等级定义

| 等级 | 五行关系 | 说明 | 处理建议 |
|------|----------|------|----------|
| 🟢 极高 | 用生体 | 失物主动来寻 | 保持平常心，很快就能找到 |
| 🟢 较高 | 比和 | 物品在常放处 | 仔细寻找平时放置的地方 |
| 🟡 中等 | 体克用 | 费力可找到 | 按方位重点寻找 |
| 🟠 较低 | 用克体 | 寻找较困难 | 扩大范围，寻求他人帮助 |
| 🔴 很低 | 体生用 | 物品可能已损坏 | 顺其自然，不要过于执着 |

## 注意事项

1. **信息完整**：尽量收集完整信息，信息越完整推演越精准
2. **位置信息关键**：当前位置、最后见到位置、房间类型等信息对推演准确性影响很大
3. **心态平和**：易经推演仅供参考，保持平和心态
4. **结合实际**：推演结果需结合实际环境进行寻找
5. **尊重信仰**：不强迫用户接受玄学解释
6. **科学辅助**：推演结果可作为寻找的辅助参考，不应完全依赖
7. **隐私保护**：不记录用户的个人信息和推演历史
8. **不制造焦虑**：避免使用"一定找不到"等绝对化表达
9. **动态调整**：如第一次寻找未果，可重新起卦推演
